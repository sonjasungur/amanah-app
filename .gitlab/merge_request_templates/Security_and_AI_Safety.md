## Security & AI Safety checklist

- [ ] Keine Secrets oder Credentials hinzugefügt
- [ ] Keine produktiven Daten in Fixtures oder Backups
- [ ] Secret-Scan lokal bestanden
- [ ] Gefährliche Dateinamen geprüft
- [ ] Neue Environment-Variablen dokumentiert
- [ ] AI-Ausgaben strukturiert validiert
- [ ] Quellenpflicht geprüft
- [ ] Prompt-Injection-Risiko geprüft
- [ ] Tool- und MCP-Rechte minimal
- [ ] Keine RED-Aktion ohne menschliche Freigabe
- [ ] Logging redigiert
- [ ] Tests für Fehler- und Unsicherheitsfälle ergänzt
- [ ] Kein Produktionsdeployment Bestandteil dieses MRs
